## What the HPC Book Provides (Notes from This Session)

Detailed notes were taken from 5 chapters. Key takeaways:

### Ch 6 (Stabilizers) -- Answers "recruitment vs. maintenance"
- **Stabilizer** = mechanism whose removal would fragment/drift/dissolve the category (perturbation test)
- "The pathways are historical; the stability is mechanistic. Both are needed."
- Uses perturbation grids: "What if mechanism X were absent?"
- For interjections: need synchronic feedback loops, not just bleaching paths
  - Prosodic isolation <-> syntactic non-integration (mutual reinforcement)
  - Non-referentiality -> loss of argument structure -> syntactic non-integration -> prosodic isolation
  - Pragmatic conventionalisation -> analogical extension -> strengthens distributional coherence

### Ch 7 (Projectibility) -- Answers circularity
- "A category just is whatever supports reliable inference across novel instances."
- **Field-relative projectibility**: same items, different HPCs for different purposes
- Partial observation of SOME cluster properties licenses inference to OTHERS -- that's projectibility
- "Interest picks the question; mechanisms determine the answer" (breaks circularity)
- Three diagnostic checks: cluster check, homeostasis check, projectibility check
- The book doesn't use "pragmatic projectibility" verbatim but the architecture supports it via field-relative projectibility

### Ch 8 (Failure Modes) -- Answers "HPC vs. prototypes" and "prediction vs. accommodation"
- Three failure modes: thin (smoke ring), fat (wastebasket), negative (complement class)
- **Dyirbal language death**: prototype theory describes edge-inward erosion; HPC explains WHY (mechanism weakening)
- Two-diagnostic test (projectibility + homeostasis) = falsifiability
- 7-step HPC-Kind Audit template directly applicable to interjections
- "Categories generate habits; classes are merely named."

### Ch 11 (Lexical Categories) -- Provides typology
- Four-part typology: robust (N/V), thin (Adj), fat label (Adv), braided (pronouns)
- Interjections should be argued as **thin but genuine** (like adjectives): real clustering, fewer mechanisms than N/V, but NOT a wastebasket
- Counterfactual template: "What if only one mechanism operated?"
- Five mechanisms for N/V; interjections will have fewer but converging ones

### Appendix (Predictions) -- Provides testable predictions
- P1 (Asymmetric fraying): tight diagnostics erode before loose ones
- P3 (Phase transition): bleaching should show a tipping point, not a smooth cline
- P5 (Conditioning recovers structure): gradience at boundaries decomposes under social conditioning
- P10 (Hub-node asymmetry): one property's removal destabilises more than others
- P11 (Competitive exclusion): *nay* > *yeah* -- survivor has greater mechanism access
- D1 (Projection without maintenance): interjections must show maintenance, not just co-occurrence
